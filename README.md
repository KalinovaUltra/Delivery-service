<br>Проект выполнен в рамках курса по аналитике. Мы разработали и обосновали новую функцию для приложения **«Самокат»** — подбор рецептов на основе продуктов, уже добавленных пользователем в корзину.

<br>**Проблема**: пользователь не знает, что приготовить из выбранных продуктов, тратит время на поиск рецептов вне приложения и часто бросает корзину.

<br>**Решение**: интерактивный режим «Составить рецепт» с выбором продуктов, показом рецептов и возможностью докупить недостающие ингредиенты одним кликом.

<br>**Основные этапы работы**
<br>Формулировка гипотез о проблеме, ценности и поведении пользователей.
<br>Проведение интервью и проверка гипотез.
<br>Формализация требований: Пользовательские истории (User Stories), функциональные требования, нефункциональные требования с приоритизацией по MoSCoW
<br>Прототипирование в Figma (переходы, экраны корзины, выбора продуктов, списка рецептов, карточки).
<br>Формирование задачи на разработку.
<br>Определение метрик успеха и способов мониторинга.
